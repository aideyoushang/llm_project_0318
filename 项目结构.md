# 项目结构（Hotel Review AI）

## 项目定位
- 酒店评论可视化分析平台 + RAG 智能问答系统
- 前后端分离：前端 Next.js（Dashboard + Chat UI），后端 FastAPI（RAG 核心逻辑与接口）
- 执行环境：所有运行与部署在云服务器完成；本地仅用于对话与协作

## 目录结构

### 前端（Next.js）
- frontend/
  - package.json：前端依赖与脚本
  - next.config.js：Next 配置
  - tsconfig.json：TS 配置与路径别名（@/*）
  - src/app/
    - page.tsx：首页（入口与导航）
    - dashboard/page.tsx：看板页（后续接图表与联动）
    - qa/page.tsx：问答页（对接后端 /api/v1/chat）
  - src/components/
    - qa/：问答相关组件（Chat、CitationBadge）
  - src/lib/
    - api.ts：HTTP 请求封装
    - qa.ts：调用后端 chat 接口的客户端逻辑

### RAG 后端（FastAPI）
- rag-service/
  - main.py：FastAPI 入口（healthz、/api/v1/chat）
  - rag_service/modules/
    - rag_system.py：管线总控（intent -> retrieve -> rerank -> generate）
    - intent.py：意图模块（是否检索、query 解析）
    - retriever.py：检索模块（后续接 BM25 + Vector + 融合）
    - ranker.py：重排模块（后续接 rerank + 多因子）
    - generator.py：生成模块（后续接 Qwen2.5-7B-Instruct 严格引用格式）
  - data/：后端离线依赖（如倒排索引、向量库等）

### 共享 Python 库（被脚本与后端复用）
- src/hotel_review_ai/
  - config.py：统一路径管理（data/models/artifacts）
  - data/：数据清洗与读写
  - rag/：索引构建与问答逻辑（后续逐步与 rag-service/modules 对齐）
  - llm/：模型加载与推理封装（后续接 Transformers + Qwen2.5）

### 数据与产物
- data/
  - raw/：原始数据落盘（当前：data/raw/tripadvisor/*.parquet）
  - cleaned/：清洗后数据（待生成）
  - rag/：RAG 语料/切分产物（待生成）
- artifacts/
  - faiss/：向量索引落盘（计划使用）
  - runs/：训练/评测运行产物（计划使用）
- models/
  - base/：基座模型本地缓存（可选）
  - adapters/：LoRA/QLoRA adapter（后续）

### 脚本
- scripts/
  - bootstrap_structure.py：创建目录骨架
  - download_tripadvisor_dataset.py：从 HF（可走 hf-mirror）下载 tripadvisor 数据集并落盘
